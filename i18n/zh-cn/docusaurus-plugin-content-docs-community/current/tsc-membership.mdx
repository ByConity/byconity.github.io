# 加入 ByConity 技术指导委员会

本指南旨在提供如何成为 ByConity 技术指导委员会（TSC）成员的信息。它概述了 ByConity 中的开发角色，TSC 的定义，成立的原因，基本原则，成为 TSC 成员的好处，以及成为 TSC 成员的步骤。

## ByConity 的开发角色

- **用户：** 使用 ByConity 软件并通过提交错误报告和功能建议来提供反馈的个人。
- **贡献者：** 向 ByConity 计划贡献代码、文档、拉取请求评审、设计或其他技术成果的社区成员。
- **提交者：** 拥有修改（“提交”）项目存储库中的源代码、文档或其他技术成果能力的贡献者。
- **维护者：** 由于其在项目开发和演进方面的贡献而被选举的提交者。他们可以访问代码存储库，可以对社区相关决策进行投票，并可以提议其他活跃贡献者成为提交者。

## 什么是技术指导委员会？

TSC 是一个由维护者组成的团队，负责维护和管理 ByConity 项目。技术指导委员会（TSC）还帮助制定项目的路线图和方向，确保其实现目标，并促进社区内的合作。

## 为什么成立 TSC

当 ByConity 在 2023 年开源时，成立 TSC 是为了确保一个开放的治理体系，促进计划中所有参与者的所有权和中立性。

TSC 成员的一些基本原则：

- 没有拥有特权的创始成员
- 所有贡献者平等
- 异步决策制定
- 每个维护者都有资格自动成为 TSC 成员

## 成为 TSC 成员的好处

作为 TSC 成员，您将享受以下好处:

- 获得 ByConity 社区的认可
- 参与涉及整个计划的议题的投票过程
- 参与计划的关键决策。

### 增减 Committer 或技术指导委员会（TSC）成员

TSC 成员可以通过投票来增减 Committer 或 TSC 成员。

### 投票

- 投票基于每个投票成员有一票的原则。
- 为了方便成员投票，例如在为新的提交者和维护者投票时，需要至少两票（不包括提名者的投票）才能算作投票成功。

### 辞职

如果某个委员会成员由于个人原因决定辞去职位，他们需要通过书面通知告知委员会。

### 撤职

TSC 成员可以通过 TSC 大多数成员的赞成票被撤职。

### 如何成为 TSC 成员

要成为 TSC 成员，您需要先成为 ByConity 项目的维护者，一旦您被投票成为 TSC 成员，您将被邀请成为 TSC 成员。
您可以通过以下方式在 ByConity 组织内成为维护者：

- 长期支持和贡献现有项目
- 由其他维护者邀请加入他们成为维护者。
