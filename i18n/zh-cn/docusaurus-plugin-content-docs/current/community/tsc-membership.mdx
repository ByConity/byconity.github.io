# 加入 ByConity 技术指导委员会

本指南旨在提供如何成为 ByConity 技术指导委员会（TSC）成员的信息。本文概述了 ByConity 中的开发角色，TSC 的定义、形成原因、基本原则、成为 TSC 成员的好处以及步骤。

# ByConity 中的开发角色

用户：使用 ByConity 软件并以错误报告和功能建议的形式提供反馈的人。

贡献者：社区成员，为 ByConity 项目贡献代码、文档、拉取请求审查、设计或其他技术成果。

提交者：已获得修改（“提交”）项目代码、文档或其他技术成果的能力的贡献者。

维护者：由于他们的功绩和对项目开发和演变的贡献而当选的提交者。他们可以写入代码库，可以投票进行社区相关决策，并可以提议其他积极的贡献者成为提交者。

# 技术指导委员会是什么？

技术指导委员会（TSC）是一组维护者，他们为ByConity项目的维护和治理做出贡献。他们负责做出影响整个项目的决策，而不仅仅是单个项目。TSC成员承诺参与投票过程。

# TSC 成立的原因

TSC 的成立是为了确保在 ByConity 在 2023 年开源时，建立一种推动所有参与者拥有权和中立性的开放治理体系。

# TSC 成员的基本原则

没有特权的创始成员

所有贡献者平等

异步决策制定

每个维护者都可以自动成为 TSC 成员

# 成为 TSC 成员的好处

获得 ByConity 社区的认可

参与涉及整个项目的投票过程

参与关键决策制定

# 如何成为 TSC 成员

任何维护者都可以成为 TSC 成员。要成为 TSC 成员，您需要成为 ByConity 项目中的一个维护者。

您可以通过以下至少一种方式成为 ByConity 组织中的维护者：

支持并提交现有项目

被其他维护者邀请加入他们作为维护者
